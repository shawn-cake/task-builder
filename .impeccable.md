## Design Context

### Users
Cake agency project managers — internal team only. Used during the workday (office or remote) to spin up Teamwork task lists from client emails or briefs. Users are familiar with Cake's workflows and Teamwork; the tool needs to be fast and frictionless, not explain itself. Not anxious — just focused.

### Brand Personality
Warm, expert, approachable. Cake is a full-service digital agency that knows its craft. The tool should feel like it was built by designers, not assembled from a template.

Three-word brand voice: **precise, warm, efficient.**

### Aesthetic Direction
Shopify Polaris-inspired: cool gray page background, white surface cards, refined professional hierarchy. Light mode only — daytime, office context. Not a consumer product; a focused internal tool. Clean and efficient without being cold or corporate.

**Anti-references**: generic SaaS dashboard grids, neon-accent dark mode, heavy nested card layouts, modal-heavy flows.

### Design Principles

1. **Transparent until interactive** — inputs and textareas have no visible border at rest. Border appears on hover; focused state adds background + border. Keeps the UI calm between edits.

2. **Labels are clearly secondary** — `--muted` color, 12px, 500 weight, fixed-width column. Content is always visually primary.

3. **Two button tiers, no more** — `.primary` (filled `--blue-dark`) for the one main action per view; `.ghost` (outlined with `--border-strong`) for back/secondary actions. Never two primaries on the same screen.

4. **Inline editing, not modals** — the preview screen edits in place with auto-expanding textareas. No dialogs for editing content.

5. **Space efficiency** — this is a task tool, not a dashboard. Dense but not cramped. Subtask rows are compact; the summary card uses tight label columns. Give content the room, not chrome.

### CSS Architecture

This project uses **Tailwind CDN** (`preflight: false`) alongside a hand-authored `styles.css`. The relationship:

- `styles.css` owns all base resets, design tokens (`:root`), and semantic component classes (`.preview-summary`, `.inline-edit`, `.subtasks`, `.badge`, etc.)
- Tailwind CDN is configured with `preflight: false` so its injected `<style>` tag doesn't override our CSS. The Tailwind config registers brand colors as utility class names (`cake-blue-dark`, `app-bg`, etc.) for ad-hoc utility use in HTML or JS-generated markup.
- **Never add Tailwind utility classes that duplicate what a semantic class already does.** Use semantic classes for components; use Tailwind utilities for one-off layout adjustments that don't belong in a named class.
